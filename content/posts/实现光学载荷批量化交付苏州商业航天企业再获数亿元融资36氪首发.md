# 诺和诺德 1.3 TB 内部数据被盗：制药巨头的网络安全警钟

2026年3月，丹麦制药巨头诺和诺德（Novo Nordisk A/S）的两个公开Web页面上的JavaScript文件，正在被每一位访问者悄然下载。这些压缩代码（minified JS）中嵌入了明文的 Azure DevOps Token 和 GitHub Personal Access Token (PAT)。任何浏览器在渲染这些页面时，都自动将这些凭证拉取到了访问者的设备中。[^1]

这不是一段虚构的开场。这是 CybelAngel 安全研究团队在回溯诺和诺德「1.3 TB 数据被盗事件」时还原的真实攻击路径。

## 事件全景：从隐蔽窃取到公开泄露

2026年6月11日，诺和诺德公开披露了一起重大安全事件，确认其「有限的内部IT系统」遭遇了未经授权的访问，并有非公开个人数据被外部复制。[^2] 此后数日，一个名为 **FulcrumSec** 的数据窃取勒索组织（Data Extortion Group）宣称对此次攻击负责，声称窃取了 **1.3 TB** 内部数据（超过70万个文件），并提出 **2500万美元** 赎金要求。[^3]

诺和诺德拒绝支付赎金后，FulcrumSec于6月15日开始在暗网陆续泄露所窃数据。更令人关注的是，第二个黑客组织 **TheUSERS007** 也声称入侵了诺和诺德的系统，索要 **5000万美元** 赎金，并宣称使用了名为「venomware」的自学习自适应AI引擎。[^4]

## 真正的入口：不是GitHub，是JavaScript

多数媒体最早报道诺和诺德事件时，猜测攻击始于 GitHub 仓库泄露。但 CybelAngel 的深入分析推翻了这一假设。

两个关键入口点分别是：嵌入在 `dev.nnedl.pub.aws.novonordisk.com` 压缩JS包中的 Azure DevOps Token，以及嵌入在 `datahub-sand.novonordisk.com` 客户端代码中的 GitHub PAT。这两个资产均为开发或沙盒环境，本不在安全团队的主动监控范围内，但已「漂移」出受控边界。[^1]

从这两个初始凭证出发，攻击链逐步展开。GitHub PAT 允许访问私有代码仓库，而仓库中发现了大量硬编码的生产环境凭证——API令牌（API Token）、数据库连接字符串、服务账号密码等。结合一个 `alg:none` JWT（JSON Web Token）认证绕过漏洞（存在于开发环境的 COSMOS 临床API中），攻击者在多个平台间完成了横向移动：GitHub、Azure DevOps、AWS、HuggingFace、Okta。[^1]

CybelAngel 的报告指出：「这些凭证并非因疏忽被推送到公共仓库——它们被Web应用在运行时直接提供给访问页面的浏览器。没有任何代码仓库扫描工具能检测到这一点。」[^1]

攻击者的驻留时间（Dwell Time）约为 **2.5 个月**。GitHub 在约2周后标记了异常活动，Azure 在约3周后标记，而 HuggingFace 和 Okta 据报告在事件披露前从未被检测到。[^1]

## 1.3 TB 数据的组成：密码可以重置，研发无法重建

FulcrumSec 公布的数据清单揭示了此次事件的战略级严重性。需要明确的是，以下数据规模来自攻击者的公开声称，尚未获得诺和诺德或第三方安全机构的独立验证：

| 数据类型 | 声称规模 |
|---------|---------|
| 总数据量 | 1.3 TB（70万+文件） |
| 代码仓库 | 4,748 个私有仓库 |
| 专有AI模型 | 33个（含1个16.7GB多模态模型检查点） |
| 训练数据集 | 约407MB生物/化学训练数据 |
| 细胞显微图像 | 494 GB |
| 员工凭证 | 163,234 条 |
| 会话cookie（Session Cookie） | 2,932 个 |
| 临床试验相关患者 | 约11,500名 |

诺和诺德官方确认了部分临床患者数据和医护人员数据受到影响，但声明患者数据是假名化的（Pseudonymized），不包含直接身份标识信息，因此第三方无法识别临床试验参与者。[^2] 这一说法与攻击者声称的「1.3 TB全量数据」之间存在差异，需等待第三方取证报告交叉验证。

Ransomnews 通过 SecurityAffairs 指出，2,932个会话cookie的暴露是「最危险的泄露类型之一」——会话cookie意味着攻击者无需密码和MFA（多因素认证）即可恢复登录会话。[^3]

TheTradable 的分析师 Artem Voloskovets 作出了一个精准的判断：「密码可以在一天内重置。但重建多年的科研成果是另一回事。」[^5] 33个专有AI模型、约50MB的 NovoPert 内部工具源代码、113个完整训练日志——这些是诺和诺德多年研发投入的凝结输出。与客户数据库不同，AI模型检查点（Model Checkpoint）在泄露后不会「失去价值」：获得训练好的多模态生物数据的竞争对手，继承的是能力，而不仅仅是记录。[^3]

## 两份赎金要求，两个黑客组织

FulcrumSec 本身值得关注——该组织首次出现在2025年9月，2026年宣称的受害者达到25个，诺和诺德是其迄今为止宣称的最大目标。该组织采用「纯数据窃取+勒索」模式，不使用勒索软件加密（Ransomware）。[^1]

而第二个黑客组织 TheUSERS007 的介入，给事件增添了更多不确定性。FiercePharma 报道称，TheUSERS007 声称通过不同的途径获得了不同的敏感数据。[^4] 如果属实，这意味着诺和诺德的攻击面可能比最早认知的更大，而不仅仅是一个JavaScript文件引发的连锁反应。

## 制药行业：正在被系统性地瞄准

诺和诺德事件并非孤立个案。制药行业正成为网络攻击的高价值目标。

2026年3月，医疗器械巨头史赛克（Stryker）遭到伊朗-linked 组织 Handala 的攻击，自称20万系统被擦除、50TB数据被窃，导致部分手术推迟。[^6] 2026年5月，West Pharmaceutical Services 遭受勒索软件攻击，全球运营中断，被迫停产隔离。[^7] 2024年2月，Change Healthcare 遭遇勒索软件攻击，约1.9亿人受影响，成为美国历史上最大的医疗数据泄露事件。[^8]

CybelAngel 的制药威胁报告显示，2025年制药行业57.5%的攻击涉及勒索软件或数据窃取——其中29.1%为勒索软件（Ransomware）攻击，26.7%为纯数据窃取（Data Exfiltration）攻击。[^9] 攻击者的商业模式正在从「锁定系统并索要解密赎金」转向「窃取那些无法轻易替代的信息」。[^5]

诺和诺德事件的特殊之处在于，它同时涉及了三类高度敏感的数据：临床试验患者信息、知识产权级AI研发成果、以及生产设施相关的 IT/OT 网络信息。TheTradable 对此作出了准确的描述：「这份泄露数据的清单读起来不像传统的数据泄露报告，而更像一份现代制药公司运营方式的地图：软件仓库、AI项目、研发管线、制造系统和临床项目。」[^5]

## 诺和诺德的应对：及格，但远远不够

客观而言，诺和诺德在事件发现后的应对是有章可循的：6月11日识别到入侵后，立即激活事件响应协议，聘请外部网络安全取证公司，通知数据保护机构和执法部门，将部分内部IT系统暂时下线以遏制损害，同时确认药物生产和供应链未受影响。[^2]

但更值得追问的是：一个市值数千亿美元的全球制药巨头，为何让两个含有明文凭证的JavaScript文件在公开可访问的Web应用中存在数月甚至数年未被发现？为何攻击者在网络中「低噪声慢速窃取」数据长达2.5个月，却没有触发任何有效告警？

CybelAngel 的研究结论指向了一个系统性的缺陷：攻击面管理（Attack Surface Management, ASM）的缺失。代码仓库扫描工具只能检测推送到仓库的凭证，却无法监控Web应用在运行时（Runtime）向浏览器提供的JavaScript内容。[^1] 这种「安全可见性缝隙」并非诺和诺德独有，而是整个行业的普遍问题。

Shieldworkz 的分析同时提出了一个合规层面的担忧：如果未来的调查发现攻击者对经过验证的 GxP（良好生产规范）环境存在未授权访问，诺和诺德可能面临额外的监管合规审查。[^10]

## 结语

诺和诺德事件的价值不在于它揭示了某个特定的技术漏洞——JavaScript中嵌入凭证、开发环境漂出监控范围、会话cookie被窃取，这些都不是新颖的攻击手法。它的价值在于，它向制药行业展示了一个正在加速到来的现实：网络犯罪已从「勒索运营」进化到「知识产权掠夺」阶段。

当攻击者不再需要加密你的文件，只需要找到你遗忘在某个角落的明文凭证，就能顺藤摸瓜搬走你的AI模型、训练数据和研发管线时，传统的「边界防御+事后响应」范式已经力不从心。对于诺和诺德而言，密码可以重置、系统可以重建，但那33个被下载的AI模型和494GB的细胞显微镜图像——那些代表着数以亿计的研发投入的数字化资产——已经无法像从未发生过一样被追回了。

这不是一个企业的失败。这是一个行业在数字化转型过程中，安全建设速度远远落后于数字化速度的现实样本。
